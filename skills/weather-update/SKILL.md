# weather-update Skill

## 目标

固定执行天气预报网站的更新流程，并用中文输出简单总结。

## 执行流程

1. 读取项目顶层 `AGENTS.md`，确认工作规则。
2. 检查项目结构是否包含必要文件和目录。
3. 检查 `.env` 是否存在。
4. 确认 API Key 只通过环境变量读取，不写死在代码中。
5. 执行 `npm run update`。
6. 检查 `data/weather.json` 是否存在并包含城市、温度、天气、更新时间。
7. 检查 `public/index.html` 是否存在并包含城市、温度、天气、更新时间。
8. 检查 `logs/update-log.md` 是否写入本次执行结果。
9. 输出中文总结。

## 失败处理

如果更新失败，必须说明：

- 哪一步失败
- 失败原因
- 建议用户如何修复
